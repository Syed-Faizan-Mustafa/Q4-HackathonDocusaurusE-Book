# Glossary

Key terms used throughout this e-book.

## A

**Action** - ROS 2 communication pattern for long-running tasks with feedback and cancellation.

## C

**Colcon** - Build tool for ROS 2 workspaces.

## D

**DDS** - Data Distribution Service, the middleware underlying ROS 2 communication.

## G

**Gazebo** - Physics-based robotics simulator integrated with ROS 2.

## I

**Isaac Sim** - NVIDIA's photorealistic robotics simulator with RTX ray tracing.

## L

**LLM** - Large Language Model, used for task planning in VLA systems.

## N

**Nav2** - ROS 2 Navigation Stack for autonomous robot navigation.

**Node** - A process in ROS 2 that performs computation.

## Q

**QoS** - Quality of Service settings for ROS 2 communication reliability.

## R

**ROS 2** - Robot Operating System 2, middleware for robotics development.

**rclpy** - ROS 2 Client Library for Python.

## S

**SDF** - Simulation Description Format, used by Gazebo.

**Service** - ROS 2 communication pattern for request/response interactions.

## T

**TF** - Transform library for tracking coordinate frames.

**Topic** - ROS 2 communication channel for publish/subscribe messaging.

## U

**URDF** - Unified Robot Description Format for robot models.

## V

**VLA** - Vision-Language-Action, AI systems that connect perception, language, and robot control.

**VSLAM** - Visual Simultaneous Localization and Mapping.

## W

**Whisper** - OpenAI's speech recognition model.
